# Quiz Multiplayer Web App
배포주소:
## Problem

로그인 기반 프로젝트(JWT, OAuth)는 각각 구현해본 경험이 있었지만,
실시간으로 상태가 변화하는 온라인 프로젝트는 아직 제대로 다뤄본 적이 없었습니다.

특히 여러 사용자가 동시에 같은 공간에 접속하고,
각 플레이어의 행동에 따라 게임 상태가 실시간으로 변경되는 구조를 직접 설계해보고 싶었습니다.

프로젝트 주제를 퀴즈 형태로 선택한 이유는 단순 CRUD보다:

- 상태 흐름 제어
- 타이머 처리
- 점수 계산
- 실시간 게임 진행
- 사용자 상호작용

등 다양한 로직을 직접 구현할 수 있었고,
직접 문제를 제작하는 과정 자체도 개발 공부에 도움이 될 것이라고 생각했기 때문입니다.

## Approach

프로젝트는 React + TypeScript 기반으로 제작했으며,
온라인 게임의 흐름을 상태 중심으로 설계하는 방향으로 개발했습니다.

### 핵심 설계 방향

- Room 기반 상태 관리 구조 설계
- 게임 상태를 status 기반으로 분리
- 컴포넌트 역할 분리
- 실제 온라인 환경을 고려한 구조 설계
- Room 상태 구조

게임 흐름 대부분이 room 상태를 기준으로 동작하도록 설계했습니다.

## Action
- 온라인 방 시스템 구현
- 방 생성 / 입장 / 퇴장 기능 구현
- 최대 인원 제한 처리
- 게임 진행 중 입장 제한
- 방장 자동 위임 기능 구현
- 실시간 게임 흐름 설계

### 방 상태관리
waiting
→ countdown
→ playing
→ result
→ finished

이를 통해:

- 시작 카운트다운
- 문제 결과 대기
- 다음 문제 전환
- 게임 종료 처리

등을 안정적으로 구현할 수 있었습니다.

### 제한 시간 및 진행 바 구현

문제별 제한 시간을 두고,
남은 시간에 따라 진행 바가 자연스럽게 감소하도록 구현했습니다.

답변 중복 제출 방지

UI와 로직 양쪽에서 중복 제출을 방지했습니다.

또한 버튼 disabled 상태를 함께 적용해 UX를 개선했습니다.

채팅 및 시스템 메시지 구현

플레이어 채팅뿐 아니라:

- 입장
- 퇴장
- 방장 변경
- 게임 시작

등을 시스템 메시지로 자동 출력하도록 구현했습니다.

### 사용자 확장성

- message
- 사용자 정의 문제집 기능 구현
- 문제집 생성
- 수정 / 삭제
- 카테고리 분류
- localStorage 저장

기능을 구현해 사용자 확장성을 높였습니다.

## Performance

프로젝트를 통해 단순 UI 구현을 넘어서,
상태 기반 게임 로직 설계 경험을 얻을 수 있었습니다.

특히 다음 부분에서 큰 성장을 경험했습니다.

- 상태 관리 구조 설계
- room 상태 중심 설계
- 게임 상태 분리
- 타이머 흐름 관리
- 컴포넌트 역할 분리
- React 로직 처리 경험
- useEffect 기반 타이머 처리
- 상태 변경 타이밍 제어
- 렌더링 흐름 이해
- 상태 업데이트 충돌 디버깅 경험
- 사용자 경험 개선
- 카운트다운 연출
- 문제 전환 템포 조절
- 진행 상태 시각화
- 채팅 및 시스템 메시지 처리

등을 통해 단순 기능 구현이 아닌 실제 게임 같은 흐름을 만드는 경험을 할 수 있었습니다.
